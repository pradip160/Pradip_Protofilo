IT Support Knowledge Base

Practical first-line troubleshooting examples written in a service desk style.

Problem: Laptop is slow to start, open apps, or respond to user actions.

Troubleshooting steps: Restart the laptop, check Task Manager, close unused apps, confirm free storage, check Windows updates, scan for malware, and test performance after login.

When to escalate: Escalate if disk, memory, malware, profile corruption, or hardware failure is suspected.

Example ticket note: User reports slow laptop. Restarted device, checked startup apps and storage, advised update completion. Escalating if issue continues after reboot and update cycle.

Problem: User cannot connect to Wi-Fi or has no internet access.

Troubleshooting steps: Check Wi-Fi status, reconnect to the network, run ping tests, use ipconfig, forget and rejoin the network, restart the router if appropriate, and test another website.

When to escalate: Escalate if multiple users are affected, DNS/DHCP appears unavailable, or network equipment may be down.

Example ticket note: User unable to access internet over Wi-Fi. Confirmed Wi-Fi enabled, reconnected network, checked IP details, and tested connectivity. Escalating if DHCP/DNS issue persists.

Problem: Print jobs do not print or remain stuck in the queue.

Troubleshooting steps: Check printer power and paper, confirm default printer, clear stuck jobs, restart the print spooler, check network connection, and reinstall the printer if approved.

When to escalate: Escalate if the printer is offline for many users, has hardware faults, or needs admin changes.

Example ticket note: User unable to print. Confirmed printer selection, checked queue, cleared failed job, and retested printing. Monitoring for repeat failures.

Problem: Outlook will not open, freezes, or shows a profile/mailbox error.

Troubleshooting steps: Restart Outlook and the device, open Outlook in safe mode, check Microsoft 365 service status, test webmail, check profile settings, and repair Office if needed.

When to escalate: Escalate if mailbox access, licensing, profile corruption, or tenant-wide service problems are suspected.

Example ticket note: Outlook not opening for user. Tested webmail, restarted app/device, and checked for profile error. Escalating if profile repair or account permission review is required.

Problem: Microphone or camera is not detected during Microsoft Teams calls.

Troubleshooting steps: Check Teams device settings, confirm Windows privacy permissions, reconnect external devices, restart Teams, update drivers, and test a Teams test call.

When to escalate: Escalate if device drivers fail, hardware is faulty, or policy settings block access.

Example ticket note: Teams camera/microphone issue. Verified selected devices, checked Windows permissions, restarted Teams, and completed test call after changes.

Problem: User cannot connect to VPN or VPN disconnects repeatedly.

Troubleshooting steps: Confirm internet access, restart VPN client, check username and password, retry MFA, test another network, and confirm the VPN client is up to date.

When to escalate: Escalate if VPN gateway, account permission, certificate, or MFA configuration needs admin review.

Example ticket note: VPN connection failing. Confirmed home internet works, restarted client, checked credentials and MFA prompt. Escalating with error details if connection still fails.

Problem: User cannot log in to Windows, email, or an application.

Troubleshooting steps: Check username format, caps lock, account lockout status, password reset process, MFA prompt, and whether other services are accessible.

When to escalate: Escalate if account permissions, identity provider, MFA reset, or security concerns require admin action.

Example ticket note: User unable to log in. Confirmed affected system, checked basic credential issues, advised password reset route, and escalated for account unlock if required.

Problem: External monitor is blank or not detected.

Troubleshooting steps: Check power and input source, reconnect cable, test another port, use Windows display settings, press Win + P, restart the laptop, and test another cable if available.

When to escalate: Escalate if the dock, graphics driver, monitor, or port appears faulty.

Example ticket note: External monitor not displaying. Checked power/input, reseated cable, used display settings, and tested duplicate/extend mode. Escalating if hardware fault remains.

Problem: User needs approved software installed or updated.

Troubleshooting steps: Check request details, confirm approval, review licence needs, install from approved source, test launch, and document version installed.

When to escalate: Escalate if admin rights, licensing, procurement, or security approval is needed.

Example ticket note: User requested software installation. Confirmed software name, business need, and approval route. Awaiting licence/admin approval before installation.

Problem: OneDrive files are not syncing or show sync errors.

Troubleshooting steps: Check OneDrive status icon, confirm sign-in, review sync errors, check storage, rename invalid files, restart OneDrive, and test sync with a small file.

When to escalate: Escalate if tenant policy, account storage, permissions, or file recovery is required.

Example ticket note: OneDrive sync issue reported. Checked sign-in, sync status, storage, and file naming errors. Restarted OneDrive and tested sync with a small file.
